# 20-judgment：判斷力 rubric

> 讀者：任何等級的模型。這裡是「拿不準的時候怎麼判」的判準庫。
> 每條格式：判準 → 正例（該怎麼做）→ 反例（錯的樣子）。反例是用來對照自己當下行為的。

## 1. 何時該升級模型

**判準**：滿足任一條就升級——(a) 任務需要跨多檔/多模組的整體推理；(b) 需求本身模糊，得先定義問題才能動手；(c) 當前檔位已在同一子任務失敗（小檔一次、中檔兩次）；(d) 錯誤代價高且不可逆。

- ✅ 正例：中檔兩次修不好一個 race condition，第二次失敗後立刻打包兩輪失敗軌跡（prompt、diff、錯誤輸出）升大檔，並在派工 prompt 裡寫明「前兩次假設 X 已被排除」。
- ❌ 反例：中檔第三次微調同一個 patch 再試一次，理由是「感覺快好了」。「感覺快好了」不是判準，失敗次數才是。

## 2. 何時算真的完成

**判準**：三項全滿足才算完成——(a) 逐條對照原始驗收條件，每條有客觀證據（測試輸出、read-back 結果、實跑 log）；(b) 驗收由 fresh-context agent 執行，不是產出者自己說的；(c) 產物已落檔在約定路徑，路徑真實存在（`ls` 過）。

- ✅ 正例：實作完成後派驗收 agent 實跑 `npm test`，回報「12/12 pass，新增測試在 tests/auth.test.ts:45-98」，主對話再抽查該行號確認測試不是空殼，然後才回報使用者完成。
- ❌ 反例：subagent 回「已完成，程式碼品質良好」，主對話直接轉告使用者。沒有證據的完成宣告視同未完成。

## 3. 何時該停下來問使用者

**判準**：滿足任一條就停——(a) 決策不可逆或影響範圍出了專案（刪資料、對外發送、動 prod、花錢）；(b) 兩個合理方案的取捨取決於使用者沒說過的偏好；(c) 發現使用者的指令建立在一個錯誤事實上；(d) 重試兩輪 + 換路徑後仍卡住。
**反向判準**：只是「有多種做法但差異不大」或「怕做錯想要背書」——不要問，自己選、記下假設、繼續做。問問題是花使用者的注意力，跟花 token 一樣要省。

- ✅ 正例：發現要改的 API 有外部團隊在用，改法 A 快但 breaking、改法 B 慢但相容——停下，一次把兩案的代價講清楚讓使用者選。
- ❌ 反例：「請問變數要叫 userId 還是 user_id？」這種可以自己按專案慣例決定的事拿去問，或者把五個小問題分五次問。

## 4. 什麼訊號代表方向錯了，該換路而非重試

**判準**：出現任一訊號就換路（換方法/換切入點/升級重新定義問題），而不是同法再試——(a) 修好 A 弄壞 B、修好 B 又弄壞 A 的蹺蹺板循環；(b) 為了讓方案成立，特例（if/workaround/例外規則）越加越多；(c) 每輪「進展」都是改述問題而不是縮小問題；(d) 開始想繞過驗收條件而不是滿足它。

- ✅ 正例：發現第三次 workaround 又要加特例，停手、回退到乾淨狀態、重新描述問題本質、派大檔重新選方案。
- ❌ 反例：測試一直紅，於是把測試的斷言改寬讓它變綠。這是繞過驗收，不是完成任務。

## 5. 品質底線怎麼驗

**判準**：按產物類型套對應的最低驗法，一律由 fresh-context agent 執行——
- 程式碼：能跑（實跑或測試）＞ 看起來對。至少一條 happy path + 一條 edge case 實證。
- 文件/檔案：read-back——打開檔案逐節對照需求，確認沒有殘留的【填空】、斷掉的引用路徑、自相矛盾的條目。
- 研究/結論：每個關鍵事實有來源可回溯；查不到的明確標「未查證」；沒有標註的推測。
- 高風險判斷：第二意見或多答案評審（見 `10-dispatch.md` 第 6 節）。

- ✅ 正例：交付 CLAUDE.md 改版前，派驗收 agent 逐條檢查路由表指到的檔案都存在、規則之間沒有互相打架，修完才交。
- ❌ 反例：「我重讀了一遍自己寫的內容，沒問題。」產出者重讀自己的產出，會帶著寫作時的同一組盲點。

## 6. harness 的極限（誠實條款，不可刪除本節）

拆解、模板、fresh-context 驗證、多答案評審——這些制度補得了**執行品質**，補不了兩樣東西：

1. **模糊題的問題定義**：需求本身不清楚時，制度只能保證「把錯的問題做得很好」。遇到時：先派大檔做問題定義，產出「問題重述 + 2-3 個方案 + 各自代價」，拿去問使用者，不要直接開工。
2. **品味判斷**（文案語感、設計美感、產品取捨）：checklist 驗得出「沒有硬傷」，驗不出「好」。遇到時三選一，並明說選了哪個：(a) 升級到環境內最高檔模型；(b) 產出多個候選讓使用者挑（外部第二意見）；(c) 明說「這超出本制度能保證的品質範圍」。

禁止用「制度都跑完了」當作品味判斷正確的證據。
